# LayoutContainer

## Overview

`LayoutContainer` is the shell used to render:

- desktop side navigation
- mobile drawer navigation
- route-aware breadcrumb
- global command palette (`Cmd/Ctrl + K`) with optional assistant mode
- child route content through `<router-view />`

The stories below focus on router metadata setup, because navigation links are generated directly from the router tree.

## Router Configurations

### Wefa Route Metadata

Uses `meta.wefa.showInNavigation`, `meta.wefa.section`, and `meta.wefa.showInCommandPalette`.

<Canvas of={LayoutContainerStories.WefaNavigationMeta} />
<Canvas of={LayoutContainerStories.WefaSectionedMeta} />

```ts
{
  path: '/dashboard',
  name: 'Dashboard',
  meta: {
    wefa: {
      title: 'Dashboard',
      icon: 'pi pi-chart-line',
      showInNavigation: true,
      showInCommandPalette: true,
      section: 'Insights',
    },
  },
}
```

### Optional Assistant Mode

You can enable a prompt-based assistant mode in the command palette by passing
`commandPalette.assistant` to `LayoutContainer`.

```ts
const assistant = {
  enabled: true,
  async generateRecipe(request) {
    return await fetch('/api/assistant/recipes/', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(request),
    }).then((response) => response.json())
  },
}
```

```vue
<LayoutContainer project-title="My Project" :command-palette="{ assistant }" />
```

### Nested Routes

```ts
  path: '/products',
  name: 'products',
  children: [
    {
      path: 'category/:id',
      name: 'category',
      children: [
        {
          path: 'item/:itemId',
          name: 'item',
        },
      ],
    },
  ],
  meta: {
    wefa: { title: 'Products', showInNavigation: true, section: 'Catalog' },
  },
}
```

## Usage

Mount `LayoutContainer` once in your root app shell (`App.vue`), like the demo:

```vue
<template>
  <LayoutContainer project-title="My Project" />
</template>

<script setup lang="ts">
import { LayoutContainer, setupDepthTracker } from '@nside/wefa/containers'

const routerViewDepth = setupDepthTracker()
</script>
```

Then keep your application routes as regular page routes (do not use `LayoutContainer` as a route component):

```ts
const routes = [
  {
    path: '/home',
    name: 'home',
    component: HomeView,
    meta: {
      wefa: {
        title: 'Home',
        icon: 'pi pi-home',
        showInNavigation: true,
        showInCommandPalette: true,
      },
    },
  },
  {
    path: '/showcase',
    name: 'showcase',
    component: ShowcaseView,
    meta: {
      wefa: {
        title: 'Showcase',
        icon: 'pi pi-bars',
        showInNavigation: true,
        showInCommandPalette: true,
      },
    },
  },
]
```
