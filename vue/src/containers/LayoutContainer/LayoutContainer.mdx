# LayoutContainer

## Overview

`LayoutContainer` is the shell used to render:

- desktop side navigation
- mobile drawer navigation
- route-aware breadcrumb
- optional route-controlled breadcrumb hiding via `meta.wefa.showBreadcrumb`
- child route content through `<router-view />`

The stories below focus on router metadata setup, because navigation links are generated directly from the router tree.

## Router Configurations

### Wefa Route Metadata

Uses `meta.wefa.showInNavigation`, `meta.wefa.section`, and optional `meta.wefa.showBreadcrumb`.

<Canvas of={LayoutContainerStories.WefaNavigationMeta} />
<Canvas of={LayoutContainerStories.WefaSectionedMeta} />
<Canvas of={LayoutContainerStories.HiddenBreadcrumb} />

```ts
{
  path: '/dashboard',
  name: 'Dashboard',
  meta: {
    wefa: {
      title: 'Dashboard',
      icon: 'pi pi-chart-line',
      showInNavigation: true,
      section: 'Insights',
      showBreadcrumb: true,
    },
  },
}
```

### Nested Routes

<Canvas of={LayoutContainerStories.MixedNestedRoutes} />

```ts
{
  path: '/products',
  name: 'products',
  children: [
    {
      path: 'category/:id',
      name: 'category',
      children: [
        {
          path: 'item/:itemId',
          name: 'item',
        },
      ],
    },
  ],
  meta: {
    wefa: { title: 'Products', showInNavigation: true, section: 'Catalog' },
  },
}
```

## Footer Slot

Use the `navigation-bottom` slot to inject application-specific footer content into both the desktop side navigation and the mobile drawer footer.

```vue
<template>
  <LayoutContainer project-title="My Project">
    <template #navigation-bottom>
      <UserProfileMenu class="w-full" />
    </template>
  </LayoutContainer>
</template>
```

## Usage

Mount `LayoutContainer` once in your root app shell (`App.vue`), like the demo:

```vue
<template>
  <LayoutContainer project-title="My Project" />
</template>

<script setup lang="ts">
import { LayoutContainer, setupDepthTracker } from '@nside/wefa/containers'

const routerViewDepth = setupDepthTracker()
</script>
```

Then keep your application routes as regular page routes (do not use `LayoutContainer` as a route component):

```ts
const routes = [
  {
    path: '/home',
    name: 'home',
    component: HomeView,
    meta: {
      wefa: {
        title: 'Home',
        icon: 'pi pi-home',
        showInNavigation: true,
      },
    },
  },
  {
    path: '/showcase',
    name: 'showcase',
    component: ShowcaseView,
    meta: {
      wefa: {
        title: 'Showcase',
        icon: 'pi pi-bars',
        showInNavigation: true,
        showBreadcrumb: false,
      },
    },
  },
]
```
