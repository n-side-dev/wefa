# Vue Router

## Overview

The Vue Router is at the center of a Vue Single Page Application (SPA).

The Router dictates what route (as visible in the browser URL) should display what component/view.

On top of that, it is possible to add custom metadata to your routes to control the flow of the application.

To make all that easier to use, WeFa provides helpers and shared route metadata typing.

## External documentation

Vue Router : https://router.vuejs.org/guide/

## Router setup

In your application, setup your router as below, importing and using helpers from WeFa.
If you're using the PBT to create your project, this is already done for you.

```ts
// src/router/index.ts
import {
  createRouter,
  createWebHistory,
  type NavigationGuardNext,
  type RouteLocationNormalized,
  type RouteLocationNormalizedLoaded,
  type RouteRecordRaw,
} from 'vue-router'
import {
  libRouteRecords,
  type WefaRouteRecordRaw,
  type WefaRouteMeta,
  defaultRouteGuards,
} from '@nside/wefa/router'

const appRoutes: RouteRecordRaw[] = [
  {
    path: '/',
    name: 'home',
    meta: {
      wefa: {
        title: 'Home',
        icon: 'pi pi-home',
        showInNavigation: true,
        showInCommandPalette: true,
        assistant: {
          docId: 'catalog.home',
        },
      } satisfies WefaRouteMeta,
    },
    component: () => import('@/views/MyHomeView.vue'),
    children: [
      // ...
    ],
  },
]

const routes: RouteRecordRaw[] = [
  // Load common routes from WeFa
  ...Object.values(libRouteRecords),

  // Override WeFa routes with your own config
  {
    ...(libRouteRecords.login as WefaRouteRecordRaw),
    props: { gdpr: false }, // e.g. disable GDPR on lib-provided Login view
  },

  ...appRoutes,
]

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes,
})

router.beforeEach(
  async (
    to: RouteLocationNormalized,
    from: RouteLocationNormalizedLoaded,
    next: NavigationGuardNext
  ) => {
    // Run the default route guards provided by the library
    defaultRouteGuards(to, from, next)

    // On top, implement your own route guards
    // ...

    // If all guards passed, proceed
    next()
  }
)

export default router
```

## Adding routes to register your own views

Once you've created a Vue Single File Component (SFC), add it to your router to make it visible in your application.

```ts
{
  path: '/',
  name: 'home',
  meta: {
      wefa: {
        title: 'Home',
        icon: 'pi pi-home',
        requiresAuth: true,
        requiresUnauth: false,
        showInNavigation: true,
        showInCommandPalette: true,
        assistant: {
          docId: 'catalog.home',
        },
      },
  },
  component: () => import('@/views/MyHomeView.vue'),
  children: [
    {
      path: 'my-nice-component',
      name: 'myNiceComponent',
      meta: {
        wefa: {
          title: 'My Nice Component',
          icon: 'pi pi-home',
        },
      },
      component: () => import('@/views/MyNiceComponent.vue'),
    },
  ],
}
```

Your component can now be accessed at `/my-nice-component` (e.g `http://localhost:5173/my-nice-component`).

## Route Metadata

WeFa route metadata is stored under `route.meta.wefa` and typed as `WefaRouteMeta`.

Children routes automatically inherit metadata from their parents and can override fields.

## Assistant Metadata

When you enable the optional `Ask` mode in the command palette, routes should expose a stable
assistant document identifier under `meta.wefa.assistant.docId`. The frontend sends this compact
route manifest to the backend, and the backend matches it against richer action docs when
generating recipes or clarification questions.

```ts
{
  path: '/catalog/products/new',
  name: 'productCreate',
  meta: {
    wefa: {
      title: 'Create Product',
      icon: 'pi pi-plus-circle',
      showInNavigation: true,
      showInCommandPalette: true,
      section: 'Commerce',
      assistant: {
        docId: 'catalog.product.create',
      },
    },
  },
}
```
